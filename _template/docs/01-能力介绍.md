# 插件能力介绍

插件通过 `ctx.api` 访问宿主 BrainPlus 提供的全部能力。每个能力由 `manifest.json` 中的 `permissions` 声明控制。

## AI

| 方法 | 说明 | 权限 |
|------|------|:--:|
| `ai.chat({ messages, system?, maxTokens? })` | 调用用户当前启用的模型生成文本 | `ai` |
| `ai.getModelName()` | 获取当前启用的模型名称 | `ai` |
| `ai.openModel(url, iconUrl?)` | 在独立窗口打开模型页面 | `ai` |

```tsx
const reply = await ai.chat({
  system: '用中文回答',
  messages: [{ role: 'user', content: '总结这段文字' }],
})
```

## 文件系统（沙箱限制）

所有路径限制在插件自身安装目录内，`../` 越界直接抛异常。

| 方法 | 说明 |
|------|------|
| `fs.readFile(path)` | 读取文本文件 |
| `fs.writeFile(path, content)` | 写入文本文件 |
| `fs.readFileBase64(path)` | 读取二进制文件（Base64） |
| `fs.exists(path)` | 检查文件是否存在 |
| `fs.listDir(path?)` | 列出目录内容 |
| `fs.stat(path)` | 获取文件大小/修改时间 |
| `fs.mkdir(path)` | 创建目录 |
| `fs.unlink(path)` | 删除文件或目录 |

## 对话框 & 文档转换

| 方法 | 说明 |
|------|------|
| `dialog.openFile({ filters? })` | 打开文件选择对话框，返回路径数组 |
| `file.convert(path)` | 将 docx/pdf/xlsx/pptx 转为 Markdown |

## 外部服务

| 方法 | 说明 | 权限 |
|------|------|:--:|
| `supabase.getClient()` | 获取 Supabase 客户端（使用用户配置） | `supabase` |
| `supabase.isConfigured()` | 检查 Supabase 是否已配置 | `supabase` |
| `cloudinary.upload(file)` | 上传图片到 Cloudinary | `cloudinary` |
| `cloudinary.isConfigured()` | 检查 Cloudinary 是否已配置 | `cloudinary` |

## 沙箱执行

| 方法 | 说明 | 权限 |
|------|------|:--:|
| `sandbox.executeJS(code, packages?)` | 在沙箱执行 JavaScript（支持 npm 包） | `sandbox` |
| `sandbox.executePython(code, packages?)` | 在沙箱执行 Python（支持 pip 包） | `sandbox` |

## 基础能力（始终可用）

| 方法 | 说明 |
|------|------|
| `storage.get(key)` / `storage.set(key, val)` | 插件独立键值存储 |
| `http.fetch(url, opts?)` | 主进程代理 HTTP 请求（绕过 CORS） |
| `ui.toast(msg, type?)` | 弹出 Toast 通知（success/error/info） |
| `plugin.getDir()` | 获取插件安装目录路径 |
| `workspace.getPaths()` | 获取工作区路径（root/output/input） |
| `workspace.openFile(path)` | 用系统默认程序打开文件 |
| `workspace.listOutputs()` | 列出输出目录中的文件 |

## AI 工具注册

插件可以在 `manifest.json` 中声明 AI 工具，自动注册到对话中：

```json
{
  "tools": [{
    "name": "my_tool",
    "description": "工具描述",
    "inputSchema": { "type": "object", "properties": { "param": { "type": "string" } } },
    "outputTemplate": "结果: ${param}"
  }]
}
```

## 宿主能力不足？

如果上述 API 不满足需求，欢迎给 BrainPlus 主项目提 PR 增加新能力，或在 `ctx.api` 开放的范围内自行扩展。
