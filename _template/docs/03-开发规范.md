# 插件开发规范

## 快速开始

```bash
# 1. 复制模板
cp -r plugins/_template plugins/my-feature

# 2. 修改 manifest.json
#    - 改 id/name/description/permissions

# 3. 编写 index.tsx
#    - export function register(ctx) { ... }

# 4. 启动开发环境
npm run dev

# 5. 在插件管理页输入路径安装:
#    /path/to/brainPlus/plugins/my-feature
```

## React/TSX 开发

插件支持完整的 React + TypeScript，可 import 宿主模块：

```tsx
import React, { useState } from 'react'
import { Button } from '@/components/ui/button'
import { Card, CardContent } from '@/components/ui/card'

export function register(ctx: any) {
  const { ai, storage, ui } = ctx.api

  const MyPage = () => {
    const [result, setResult] = useState('')
    return (
      <Card>
        <CardContent>
          <Button onClick={async () => {
            const reply = await ai.chat({ messages: [{ role: 'user', content: 'Hello' }] })
            setResult(reply)
          }}>调用 AI</Button>
          {result && <p>{result}</p>}
        </CardContent>
      </Card>
    )
  }

  ctx.registerNav({ id: 'my-feature', label: '我的功能', icon: 'Package', order: 90 })
  ctx.registerRoute('my-feature', () => Promise.resolve({ default: MyPage }))
}
```

### 可 import 的宿主模块

| 路径 | 提供 |
|------|------|
| `react` | useState, useEffect 等全部 React |
| `lucide-react` | 全部图标 |
| `@/components/ui/button` | Button |
| `@/components/ui/input` | Input |
| `@/components/ui/badge` | Badge |
| `@/components/ui/card` | Card, CardHeader, CardContent |
| `@/components/ui/label` | Label |
| `@/components/ui/switch` | Switch |
| `@/lib/utils` | cn() |

## 测试

### API 自检

在插件中加入自检面板，参考 hello-world 的 `runTests()`：
- 测试每个权限对应的 API 是否可用
- 输出 pass/fail/skip 状态
- 安装后一键验证所有能力通路

### 调试

- 打开 DevTools（`Cmd+Opt+I`）查看 Console 日志
- 搜索 `[PluginSystem]` 查看插件加载状态
- 搜索 `[ErrorBoundary]` 查看渲染异常
- 插件编译错误会出现在主进程终端

### 常见问题

| 现象 | 排查 |
|------|------|
| 页面空白 | 看 Console 有没有 `[ErrorBoundary]` 日志 |
| API 不可用 | 检查 manifest.json permissions |
| 修改不生效 | 卸载后重装（插件不会热更新） |
| `@/` 导入报错 | 重启应用，确认 esbuild external 列表包含该路径 |

## 提交与推送

### 建议的目录结构

```
plugins/my-feature/
  manifest.json       ← 必须
  index.tsx           ← React/TSX 入口（必须）
  package.json        ← 必须
  supabase/           ← SQL（如有）
    └── *.sql
  docs/               ← 文档（可选）
```

### 提交建议

- 功能完整可运行
- `manifest.json` 权限按实际需要
- 包含 `package.json`（即使无依赖）
- 有数据库表的话 SQL 放 `supabase/`
- 建议复杂插件附带 `docs/`

### 母体能力不足？

如果 `ctx.api` 提供的能力无法满足需求，欢迎给 BrainPlus 主仓库提 PR：

1. 在 `src/lib/pluginTypes.ts` 定义新 API 类型
2. 在 `src/lib/pluginSystem.ts` 的 `buildPluginAPI()` 实现
3. 如需要主进程支持，在 `electron/main.ts` 加 IPC handler
4. 更新 hello-world 自检面板覆盖新 API
